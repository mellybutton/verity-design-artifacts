## TICKET 2: Generate Spec Dispatch Pipeline (Notion + Slack + Jira)

**Title:** Generate Spec dispatch pipeline: Notion + Slack + Jira integration  
**Type:** Task  
**Project:** DEV  
**Assignee:** Hilary  
**Priority:** High  
**Estimate:** 13 pts  
**Due Date:** March 26, 2026 (pilot Day 8)

**Description:**

Build the backend dispatch pipeline for Generate Spec. When Jordan clicks "Send Specs" in the modal, this pipeline:
1. Creates Notion pages from templates (Product + Engineer)
2. Sends Slack notifications to Alex + Riley
3. Creates a Jira task assigned to Riley
4. Archives any prior spec version
5. Writes Decision Log entry

### What to build

**Endpoint:** `POST /api/specs/generate`

**Request body:**
```json
{ "cluster_id": "uuid", "user_id": "user-uuid" }
```

**Pipeline steps (in transaction):**

1. **Archive old spec (if exists)**
   - Query: `SELECT spec_generation_id FROM spec_generations WHERE cluster_id = ? AND status = 'Dispatched' ORDER BY version DESC LIMIT 1`
   - If exists: `UPDATE spec_generations SET status = 'Superseded' WHERE spec_generation_id = ?`

2. **Create Notion pages**
   - Product Spec:
     - Load template from: `notion-product-spec-template.md`
     - Populate variables: [CLUSTER_NAME], [DOMAIN], [SIGNAL_STRENGTH], [DATE], [VERSION], [N], [rule list with IF/THEN/BECAUSE]
     - Create Notion page under "Generated Specs" database
     - Set metadata: Cluster relation, Spec Type: "product", Generated By: user_id
     - Return page_id + URL
   
   - Engineer Spec:
     - Load template from: `notion-engineer-spec-template.md`
     - Populate variables: same as above, plus YAML rule schema with: rule_id, priority (by specificity+manual priority), if_condition, then_action, except_clause, owner, confidence, because_reasoning
     - Create Notion page under "Generated Specs" database
     - Set metadata: Cluster relation, Spec Type: "engineer", Generated By: user_id
     - Return page_id + URL

3. **Create spec_generations database rows**
   - Insert 2 rows: one per spec_type (product, engineer)
   - Fields: cluster_id, spec_type, version (auto-increment), notion_page_id, generated_by, conflict_check_passed=true, status='Dispatched', created_at=now()

4. **Create Jira task**
   - Assigned to: Riley (557058:ee5620ac-19b6-4c6f-88d5-ed65471536e1)
   - Priority: High (if signal_strength=Strong) else Medium
   - Summary: "Implement rules from [CLUSTER_NAME] spec"
   - Description: See `generate-spec-integration-spec-v1.md` Step 3.4 for exact payload
   - Due date: +7 days from now
   - Store task key in: spec_generations.jira_task_id

5. **Send Slack notifications**
   - To Alex (DM or #eng-planning): "🚀 Product Spec: [CLUSTER_NAME]" + link to Notion + link to cluster detail (SF-005)
   - To Riley (DM or #engineering): "🚀 Engineer Spec: [CLUSTER_NAME]" + link to Notion + link to Jira task
   - Store message timestamps in: spec_generations.slack_msg_ts

6. **Write Decision Log entry**
   - Area: "Product"
   - Title: "Generated spec for [CLUSTER_NAME]"
   - Content: "Conflict check: 0 unresolved. Product + Engineer specs dispatched to Alex and Riley. Jira task DEV-###. Notion docs linked. Version [N]."

7. **Return response**
   ```json
   {
     "spec_generation_id": "uuid",
     "version": 2,
     "notion_product_link": "...",
     "notion_engineer_link": "...",
     "jira_task_id": "DEV-123",
     "slack_messages": { "alex_ts": "...", "riley_ts": "..." },
     "dispatched_at": "2026-03-19T15:30:00Z"
   }
   ```

### Error handling

- If Notion creation fails: return 500, don't create Jira or send Slack
- If Jira creation fails: Notion already created, log error, return error to UI
- If Slack send fails: log error, skip notification (spec still created)

### Files to create

- `spec_generations` table (see schema in `generate-spec-integration-spec-v1.md`)
- `/api/specs/generate` endpoint (Supabase Edge Function or Next.js API route)
- Notion template loader (read `notion-product-spec-template.md` + `notion-engineer-spec-template.md`)
- Jira API integration (use Atlassian REST API)
- Slack integration (use Slack Web API for DMs + structured messages)

### Files to update

- Backend API routes
- Database schema (add `spec_generations` table)

### Acceptance criteria

- [ ] `/api/specs/generate` endpoint works end-to-end
- [ ] Notion Product Spec page created with correct template variables
- [ ] Notion Engineer Spec page created with correct template variables and YAML schema
- [ ] spec_generations rows inserted with correct metadata
- [ ] Old spec marked Superseded if second generation
- [ ] Jira task created, assigned to Riley, with correct description
- [ ] Slack notification sent to Alex with Notion link + cluster link
- [ ] Slack notification sent to Riley with Notion link + Jira link
- [ ] Decision Log entry written
- [ ] On second generation: version increments (v2, v3, etc.), old spec archived
- [ ] Error handling: 500 if Notion fails, log errors for Jira/Slack failures
